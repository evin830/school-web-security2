# school-web-security2
2차 수정
주제 (이건 확실하지 않아서 안적음)

동기
평소 웹에서 여러 사용자가 동시에 대화하는 서비스가 어떤 방식으로 동작하는지 궁금했다. 특히 새로고침 없이 실시간으로 메시지가 전달되는 기능을 직접 구현해 보고 싶었고, 프론트엔드와 백엔드를 함께 경험해 보기 위해 실시간 채팅 웹사이트를 제작하게 되었다. 또한 Render를 이용해 직접 배포하면서 실제 인터넷 환경에서 서비스를 운영하는 과정도 경험하고자 했다. 

과정
먼저 HTML과 CSS를 이용하여 채팅 화면과 사용자 인터페이스를 구성하였다. JavaScript를 사용해 닉네임 입력, 메시지 입력, 온라인 사용자 목록, 링크 자동 변환 등의 기능을 구현하였다.
실시간 통신을 위해 Socket.IO를 사용하여 사용자가 접속하면 닉네임을 등록하고, 메시지를 서버를 통해 모든 사용자에게 전달하도록 구현하였다. Express를 이용해 정적 파일을 제공하고, Node.js 서버를 구축하였다.
메시지는 CryptoJS를 이용하여 AES 방식으로 암호화한 후 전송하고, 수신한 사용자가 다시 복호화하도록 구현하였다.
마지막으로 프로젝트를 GitHub에 업로드하고 Render와 연동하여 실제 인터넷에서 접속 가능한 웹사이트로 배포하였다. 배포 과정에서 환경 변수 포트(process.env.PORT)를 사용하도록 수정하여 Render 환경에서도 정상적으로 실행되도록 구성하였다.
결론
이번 프로젝트를 통해 단순한 웹페이지 제작을 넘어 클라이언트와 서버가 실시간으로 통신하는 구조를 이해할 수 있었다. 또한 웹 서비스를 직접 배포하면서 개발 환경과 실제 운영 환경의 차이를 경험하였고, 하나의 서비스를 처음부터 끝까지 구현하는 과정을 익힐 수 있었다. 

어려웠던 점
가장 어려웠던 부분은 로컬에서는 정상적으로 동작하던 프로그램이 Render에 배포했을 때 제대로 실행되지 않았던 점이었다. 서버 포트 설정과 Socket.IO 연결 문제를 해결해야 했으며, 클라이언트와 서버 간 데이터가 정상적으로 전달되지 않는 원인을 찾는 데 시간이 걸렸다. 또한 여러 사용자가 동시에 접속하는 상황에서 사용자 목록을 관리하는 부분도 처음에는 구현이 쉽지 않았다. 

해결
Render의 실행 환경에 맞게 process.env.PORT를 사용하도록 서버를 수정하고, Express와 Socket.IO의 설정을 다시 점검하였다. 또한 접속과 종료 이벤트를 이용하여 사용자 목록을 관리하도록 구현하여 접속자가 변경될 때마다 목록이 자동으로 갱신되도록 개선하였다. 오류가 발생할 때는 브라우저 개발자 도구와 서버 로그를 확인하면서 원인을 하나씩 찾아 수정하였다. 

성장
이번 프로젝트를 통해 HTML, CSS, JavaScript뿐만 아니라 Node.js, Express, Socket.IO를 활용한 웹 서버 개발 과정을 경험할 수 있었다. 또한 GitHub와 Render를 이용한 배포 과정까지 직접 수행하면서 실제 웹 서비스 개발의 전체 흐름을 이해하게 되었다. 기능 구현뿐만 아니라 보안과 서버 구조의 중요성도 알게 되었으며, 앞으로는 로그인 기능, 데이터베이스 연동, 서버 측 인증 등 더욱 완성도 높은 웹 서비스를 개발해 보고 싶다는 목표가 생겼다.
